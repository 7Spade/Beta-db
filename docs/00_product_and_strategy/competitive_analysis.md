# 競品分析 v2.0 - 功能矩陣對比

## 概述

本文件旨在分析營造業專案管理軟體市場的現有競爭格局，識別主要競爭對手，並評估其優劣勢。v2.0 版本深入到核心功能矩陣，以幫助我們明確 Beta-db 的市場定位和差異化策略。

## 主要競爭對手

1.  **Procore**:
    *   **定位**: 市場領導者，一體化平台，服務大中型總承包商。
    *   **優勢**: 功能全面，生態成熟，財務工具強大，市場佔有率高。
    *   **劣勢**: 價格昂貴，系統複雜，學習曲線陡峭，對小型團隊不夠靈活。

2.  **Autodesk Construction Cloud (ACC)**:
    *   **定位**: 以 BIM 為核心，打通設計到施工全流程。
    *   **優勢**: 與 Autodesk 設計軟體（Revit, AutoCAD）無縫整合，BIM 模型協同能力無與倫比。
    *   **劣勢**: 功能分散在不同模組，非 Autodesk 生態使用者整合不便。

3.  **本土傳統軟體 (例如：營造速記、高益)**:
    *   **定位**: 深耕本地市場，專注於會計和計價流程。
    *   **優勢**: 符合台灣法規和會計習慣，價格低。
    *   **劣勢**: 技術老舊，缺乏雲端與移動協作能力，UI/UX 過時。

## 核心功能矩陣對比

| 功能域 | Procore | Autodesk Construction Cloud (ACC) | 本土傳統軟體 | Beta-db 差異化策略 |
| :--- | :--- | :--- | :--- | :--- |
| **專案與任務管理** | 功能強大，支援 WBS，但設定複雜，流程較死板。 | 基礎任務管理，強項在於與模型的關聯。 | 功能非常基礎或缺乏，依賴人工和紙本。 | **簡潔與自動化**：提供無限層級任務，但透過 **AI 智慧分解** 簡化 WBS 創建。**任務委派與驗收** 流程是我們的核心閉環。 |
| **財務與成本控制** | 極度強大，包含完整預算、成本、合約、發票管理。是其核心護城河。 | 較弱，主要依賴與第三方財務軟體的整合。 | **核心優勢**，但流程僵化，與其他系統隔離。 | **另闢蹊徑**：我們不直接競爭其完整財務功能。而是專注在**進度計價**，將**已驗收的工程任務**作為計價的唯一依據，實現數據驅動的計價流程。 |
| **文件與圖紙管理** | 成熟的文件管理系統，支援版本控制和線上審閱。 | **絕對優勢**。與 BIM 360 深度整合，圖紙和模型管理是其核心。 | 非常薄弱，基本是本地文件管理，無雲端協同。| **AI 賦能**：我們的「雲端硬碟」是基礎，但核心差異在於 **DocuParse**，能從非結構化文件中自動提取工料清單，直接生成專案與合約。 |
| **現場協作** | 提供 RFI、Submittals、工地日報等完整模組。 | 同樣提供完整模組，並強調與模型的關聯。 | 幾乎沒有。現場協作完全依賴 LINE 和電話。 | **專注高頻場景**：初期我們專注在「工地日報」和「進度回報」這兩個最高頻的**快捷操作**，並確保其在移動端的體驗極致簡單。 |
| **AI 與智慧分析** | 開始引入 AI，主要用於數據洞察和風險預測。 | 也在探索 AI，例如在圖紙比對和安全分析上。 | 完全沒有。 | **AI 原生**：AI 是我們平台的核心，而非附加功能。從文件解析、任務分解到排程建議，AI 將貫穿整個核心工作流程，成為使用者的智慧助手。 |
| **開放性與整合** | 提供 API，但生態系主要圍繞大型財務軟體。 | API 相對開放，但最佳體驗仍限於其自身生態。 | 封閉系統，無整合能力。 | **事件驅動架構**：我們的架構天生就是為了整合而生。任何核心操作（如任務完成、使用者註冊）都可以廣播事件，輕鬆與其他系統對接。 |

## 結論與策略

-   **市場切入點**: 我們的目標是**為中小型營造團隊設計的、以 AI 協作為核心的現代管理平台**。
-   **核心競爭力**:
    1.  **AI 驅動的自動化**: 在文件解析、任務分解等環節，提供對手所不具備的自動化能力。
    2.  **流程創新**: 打造「任務委派 → 線上驗收 → 數據計價」的無縫閉環，解決行業核心痛點。
    3.  **極致的使用者體驗**: 專注於簡化高頻操作，提供比傳統軟體更直觀、更易用的介面。

我們的策略不是在功能數量上與 Procore 正面競爭，而是在**工作效率**和**智慧化**上建立代差優勢。
