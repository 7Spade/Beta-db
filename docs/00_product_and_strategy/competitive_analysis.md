# 競品分析

## 概述

本文件旨在分析營造業專案管理軟體市場的現有競爭格局，識別主要競爭對手，並評估其優劣勢。這將幫助我們明確 Beta-db 的市場定位和差異化策略。

## 主要競爭對手

1.  **Procore**:
    *   **優勢**: 市場領導者，功能全面，涵蓋從投標到專案結束的全生命週期。生態系成熟，與多種財務軟體整合良好。
    *   **劣勢**: 價格昂貴，對於中小型營造商門檻較高。系統複雜，學習曲線陡峭。介面較為傳統。
    *   **啟示**: 我們可以專注於提供更簡潔、更易於上手的使用者體驗，並以更具競爭力的價格切入中小型企業市場。

2.  **Autodesk Construction Cloud (ACC)**:
    *   **優勢**: 與 Autodesk 的設計軟體（如 Revit, AutoCAD）無縫整合，在 BIM 模型協作方面具有巨大優勢。
    *   **劣勢**: 功能仍較分散在不同模組中（如 Build, BIM Collaborate, Takeoff）。非 Autodesk 生態系的使用者可能會感到整合不便。
    *   **啟示**: 我們的平台應強化數據的互通性，不僅僅是設計數據，更要打通現場施工數據和財務數據。

3.  **本土傳統軟體 (例如：營造速記、高益)**:
    *   **優勢**: 深耕台灣市場，非常貼近本地的法規、會計和計價習慣。價格相對較低。
    *   **劣勢**: 技術架構老舊，大多為桌面應用，缺乏雲端協作能力。UI/UX 過時，行動端支援差。
    *   **啟示**: 我們的核心優勢在於**雲端原生**和**現代化的使用者體驗**。我們需要保留對本地化流程的支援，但用更先進的技術來實現。

## 市場機會與差異化策略

- **AI 驅動的決策輔助**: 現有競品大多停留在「數據記錄」。我們可以利用 Genkit AI，在「數據洞察」和「智慧建議」（如成本預測、排班建議）上建立核心差異。
- **專注於協作流程**: 許多軟體功能強大但流程死板。我們應專注於打通「任務委派-驗收-計價」這一核心協作閉環，提升跨組織的工作效率。
- **開放與整合**: 提供靈活的 API 和事件驅動架構，讓平台能輕易地與其他系統（如 HR、財務）對接。

---
**結論**: Beta-db 的市場切入點是**為中小型營造團隊設計的、以雲端協作為核心、具備 AI 輔助能力的現代化專案管理平台**。我們的競爭優勢不在於功能的堆砌，而在於**流暢的體驗、智慧化的數據和開放的生態**。
