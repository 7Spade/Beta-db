# 竞品分析 v2.0 - 功能矩阵对比

## 概述

本文件旨在分析营造业专案管理软体市场的现有竞争格局，识别主要竞争对手，并评估其优劣势。v2.0 版本深入到核心功能矩阵，以帮助我们明确 Beta-db 的市场定位和差异化策略。

## 主要竞争对手

1.  **Procore**:
    *   **定位**: 市场领导者，一体化平台，服务大中型总承包商。
    *   **优势**: 功能全面，生态成熟，财务工具强大，市场占有率高。
    *   **劣势**: 价格昂贵，系统复杂，学习曲线陡峭，对小型团队不够灵活。

2.  **Autodesk Construction Cloud (ACC)**:
    *   **定位**: 以 BIM 为核心，打通设计到施工全流程。
    *   **优势**: 与 Autodesk 设计软件（Revit, AutoCAD）无缝整合，BIM 模型协同能力无与伦比。
    *   **劣势**: 功能分散在不同模组，非 Autodesk 生态用户整合不便。

3.  **本土传统软体 (例如：营造速记、高益)**:
    *   **定位**: 深耕本地市场，专注于会计和计价流程。
    *   **优势**: 符合台湾法规和会计习惯，价格低。
    *   **劣势**: 技术老旧，缺乏云端与移动协作能力，UI/UX 过时。

## 核心功能矩阵对比

| 功能域 | Procore | Autodesk Construction Cloud (ACC) | Beta-db 差异化策略 |
| :--- | :--- | :--- | :--- |
| **专案与任务管理** | 功能强大，支持 WBS，但设定复杂，流程较死板。 | 基础任务管理，强项在于与模型的关联。 | **简洁与自动化**：提供无限层级任务，但透过 **AI 智慧分解** 简化 WBS 创建。**任务委派与验收** 流程是我们的核心闭环。 |
| **财务与成本控制** | 极度强大，包含完整预算、成本、合约、发票管理。是其核心护城河。 | 较弱，主要依赖与第三方财务软件的整合。 | **另辟蹊径**：我们不直接竞争其完整财务功能。而是专注在**进度计价**，将**已验收的工程任务**作为计价的唯一依据，实现数据驱动的计价流程。 |
| **文件与图纸管理** | 成熟的文件管理系统，支持版本控制和线上审阅。 | **绝对优势**。与 BIM 360 深度整合，图纸和模型管理是其核心。 | **AI 赋能**：我们的「雲端硬碟」是基础，但核心差异在于 **DocuParse**，能从非结构化文件中自动提取工料清單，直接生成专案与合约。 |
| **现场协作** | 提供 RFI、Submittals、工地日報等完整模組。 | 同样提供完整模組，并强调与模型的关联。 | **专注高频场景**：初期我们专注在「工地日報」和「進度回報」这两个最高频的**快捷操作**，并确保其在移动端的体验极致简单。 |
| **AI 与智慧分析** | 开始引入 AI，主要用于数据洞察和风险预测。 | 也在探索 AI，例如在图纸比对和安全分析上。 | **AI 原生**：AI 是我们平台的核心，而非附加功能。从文件解析、任务分解到排程建议，AI 将贯穿整个核心工作流程，成为使用者的智慧助手。 |
| **开放性与整合** | 提供 API，但生态系主要围绕大型财务软件。 | API 相对开放，但最佳体验仍限于其自身生态。 | **事件驱动架构**：我们的架构天生就是为了整合而生。任何核心操作（如任务完成、使用者注册）都可以广播事件，轻松与其他系统对接。 |

## 结论与策略

-   **市场切入点**: 我们的目标是**为中小型营造团队设计的、以 AI 协作为核心的现代管理平台**。
-   **核心竞争力**:
    1.  **AI 驱动的自动化**: 在文件解析、任务分解等环节，提供对手所不具备的自动化能力。
    2.  **流程创新**: 打造「任务委派 → 线上验收 → 数据计价」的无缝闭环，解决行业核心痛点。
    3.  **极致的用户体验**: 专注于简化高频操作，提供比传统软件更直观、更易用的介面。

我们的策略不是在功能数量上与 Procore 正面竞争，而是在**工作效率**和**智慧化**上建立代差优势。
